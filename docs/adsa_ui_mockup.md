## Design Philosophy

### Core Principles
1. **Workflow-First Design**: Users select their experiment type before accessing analysis tools
2. **Context-Specific Interface**: Each experiment type shows only relevant parameters
3. **Visual Clarity**: Consistent spacing, clear hierarchy, guided user flow
4. **Progressive Disclosure**: Advanced options hidden until needed
5. **Immediate Feedback**: Real-time validation and status indicators

### Color Scheme
- **Primary Background**: `#2B2B2B` (dark gray)
- **Secondary Background**: `#353535` (lighter gray)
- **Accent Blue**: `#4A90E2` (actions, highlights)
- **Success Green**: `#5CB85C` (calibrated, valid)
- **Warning Orange**: `#F0AD4E` (needs attention)
- **Error Red**: `#D9534F` (invalid, errors)
- **Text Primary**: `#E8E8E8` (main text)
- **Text Secondary**: `#A0A0A0` (labels, hints)

#### Card Hover State
- Subtle elevation increase (shadow)

- Immediate transition to specialized main window
- Save selection as default (optional setting)
- Load appropriate templates and defaults

- Single-click to open
- Right-click context menu: Open, Delete, Show in folder, Properties
- Shows experiment type icon next to filename
- Date formatted as relative time

- **Tab**: Navigate between cards
- **Enter**: Select highlighted card
- **Ctrl+O**: Open project dialog
- **Escape**: Exit application (with confirmation)

## Implementation Notes

### Technology Stack Recommendations

#### Desktop Application
- **Framework**: Qt 6.x (PyQt6 for Python, Qt C++ for performance)
- **Advantages**: Native look, excellent graphics, cross-platform
- **Image Processing**: OpenCV, scikit-image
- **Plotting**: Matplotlib, PyQtGraph

#### Modern Web-Based (Alternative)
- **Frontend**: React + TypeScript
- **UI Framework**: shadcn/ui or Material-UI
- **Canvas**: Fabric.js or Konva.js for image manipulation
- **Desktop**: Electron wrapper for desktop distribution
- **Advantages**: Easier updates, modern UI, collaborative features

### Key Technical Considerations

#### 1. State Management
```python
# Experiment state structure
experiment_state = {
    'type': 'sessile' | 'pendant' | 'tilted_sessile',
```

#### 2. Image Processing Pipeline
```python
def analyze_drop(image, experiment_type, parameters):
    """
    Main analysis pipeline
    """
    # 1. Preprocess
    preprocessed = preprocess_image(image)
    
    # 2. Edge detection
    edges = detect_edges(preprocessed, parameters['edge_sensitivity'])
    
    # 3. Contour extraction
    contour = extract_contour(edges)
    
    # 4. Type-specific analysis
    if experiment_type == 'sessile':
        results = analyze_sessile(contour, parameters)
    elif experiment_type == 'pendant':
        results = analyze_pendant(contour, parameters)
    elif experiment_type == 'tilted_sessile':
        results = analyze_tilted(contour, parameters)
    
    # 5. Quality check
    results['confidence'] = calculate_confidence(results)
    
    return results
```

#### 5. Error Handling
```python
class ADSAError(Exception):
    """Base exception for ADSA"""
    pass

class CalibrationError(ADSAError):
    """Raised when calibration is missing or invalid"""
    pass

class ImageLoadError(ADSAError):
    """Raised when image cannot be loaded"""
    pass

class AnalysisError(ADSAError):
    """Raised when analysis fails"""
    def __init__(self, message, suggestion=None):
        self.message = message
        self.suggestion = suggestion
        super().__init__(self.message)

# Usage
try:
    results = analyze_drop(image, exp_type, params)
except CalibrationError:
    show_notification("error", 
        "Calibration required",
        "Please calibrate before analyzing",
        action_button=("Calibrate Now", open_calibration_wizard))
except AnalysisError as e:
    show_notification("error",
        f"Analysis failed: {e.message}",
        e.suggestion if e.suggestion else None)
```

#### 6. Performance Optimization
- **Lazy loading**: Only load visible results in table
- **Image caching**: Keep processed images in memory (with size limit)
- **Background processing**: Run analysis in separate thread
- **Progressive rendering**: Show preview while processing large batches
- **Vectorized operations**: Use NumPy for all array operations

#### 7. Accessibility
- High contrast mode option
- Keyboard-only navigation support
- Screen reader compatibility
- Adjustable font sizes
- Color-blind friendly color schemes (option)

### UI/UX Best Practices Applied

1. **Consistency**: Same patterns across all experiment types
2. **Feedback**: Immediate visual response to all actions
3. **Prevention**: Disable invalid actions (grayed out buttons)
4. **Recovery**: Easy undo, clear error messages with solutions
5. **Efficiency**: Keyboard shortcuts, batch operations, templates
6. **Learnability**: Progressive disclosure, tooltips, integrated help
