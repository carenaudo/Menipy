# GUI Design

This document outlines the user interface architecture and components using PySide6.

## 1. Framework Choice

- **PySide6**: Official Qt for Python bindings, used for all UI components.

## 2. Main Window Layout

The application features a modern, pipeline-driven UI built around a top
workflow bar and split workbench:

- **Workflow Bar (Top)**: Hosts compact analysis-type buttons, source-mode
  buttons, the current source selector, export controls, and panel visibility
  toggles. The analysis buttons choose the active pipeline (`sessile`,
  `pendant`, `oscillating`, `capillary_rise`, or `captive_bubble`).

- **Setup Rail (Left)**: Hosts setup controls such as calibration, phase
  properties, pipeline-specific settings, and the main **Run Analysis** action.
  Advanced SOP/stage controls are opened through the Advanced workflow dialog
  rather than expanded inline.

- **Scientific Step Test Rail (Left, Science Focus Only)**: When the user
  chooses **View -> Focus -> Science**, a checkable **Test** button appears in
  the workflow bar. Activating it replaces the setup rail with the Step Test
  panel. The panel lists testable pipeline stages except `acquisition`, runs a
  selected stage with prerequisites, and reports timings/results/warnings.

- **Preview Panel (Center)**: A large panel that displays the source image or
  camera feed. It uses a custom `ImageView` widget to render the image and any
  graphical overlays (e.g., detected contours, ROIs, fitted lines) generated by
  the analysis pipeline.

- **Results and Diagnostics (Right/Bottom)**: Displays key metrics, results
  tables, residuals, timings, and logs. Focus presets control which panels are
  emphasized for measurement, science review, or result analysis.

This design is highly flexible, as the UI dynamically adapts to the stages and
parameters of whichever pipeline the user selects.

## 3. Configuration Dialogs

Menipy utilizes dedicated dialogs for configuring various pipeline stages,
accessible from the setup rail, Advanced dialog, or the Science-mode Step Test
panel. These dialogs provide granular control over parameters.

-   **Preprocessing Configuration Dialog**: Allows users to adjust settings for image cropping, resizing, filtering, background subtraction, and normalization.
-   **Edge Detection Configuration Dialog**: Provides extensive controls for edge detection algorithms (Canny, Threshold, Sobel, Active Contour, etc.) and contour refinement.
-   **Geometry, Physics, and Overlay Dialogs**: Exposed to the Step Test panel as
    sandboxed editors for scientific review. Sandbox edits affect test runs only
    until the user clicks **Apply**.

## 4. Scientific Step Test Panel

The Step Test panel is implemented by `PipelineStepTestPanel` and
`PipelineStepTestController`. It is intentionally available only in the Science
focus preset so routine measurement workflows remain compact.

- Stage lists are generated from the selected pipeline's `build_plan()` output,
  with `acquisition` removed.
- Test runs call `PipelineController.test_stage()`, which delegates to
  `run_with_plan(only=[stage], include_prereqs=True)`.
- The test runner silently runs `AutoCalibrator` against the selected source and
  merges detected prerequisites into the run context.
- Sandbox settings are cloned from live controllers. **Apply** writes them back
  to live settings; **Discard** reloads the live settings.

## 5. Image Display & Overlays

- The `ImageView` widget, a subclass of `QGraphicsView`, is used for all image rendering.
- Overlays like contours, points, and lines are drawn as `QGraphicsPathItem` or similar objects on the scene, which allows them to be toggled and updated without re-rendering the base image.

## 6. Saving & Export

- **Save Annotated Image**: Functionality is provided to save the current view from the Preview Panel, including all overlays.
- **Export Results**: The data from the Results Panel can be exported, typically to a CSV file.
